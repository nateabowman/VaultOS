# VaultOS Wallpaper Creation Guidelines

## Directory Structure

Wallpapers are organized by theme:
- `pipboy/` - Pip-Boy green themed wallpapers
- `vaulttec/` - Vault-Tec blue/gold themed wallpapers
- `patterns/` - Retro-futuristic patterns and abstract designs

## Required Resolutions

Each wallpaper should be provided in multiple resolutions:
- **1920x1080** (Full HD) - Most common
- **2560x1440** (QHD) - High-end displays
- **3840x2160** (4K UHD) - Ultra-high resolution

## Color Palettes

### Pip-Boy Theme
- **Primary Green**: `#00FF41` - Pip-Boy green
- **Background**: `#000000` - Black
- **Dark Green**: `#003300` - Shadows and depth
- **Light Green**: `#66FF99` - Highlights
- **Mid Green**: `#00AA2A` - Secondary elements

### Vault-Tec Theme
- **Gold**: `#FFD700` - Primary accent
- **Blue**: `#1E3A8A` - Vault-Tec blue
- **Yellow**: `#FFFF41` - Highlights
- **Dark Blue**: `#0F1F4A` - Shadows
- **Light Blue**: `#3B5FA8` - Secondary elements

## Wallpaper Types

### Pip-Boy Interface Backgrounds
- Mimic Pip-Boy 3000 interface aesthetic
- Grid patterns, scan lines, or terminal-style layouts
- Monochrome green on black
- Retro CRT/monitor feel
- Optional: HUD elements, borders, frames

### Vault Door Images
- Vault door designs (inspired, not copyrighted)
- Circular door patterns
- Vault-Tec branding elements
- Industrial/metal textures
- Lighting effects (golden/blue glow)

### Retro-Futuristic Patterns
- Atomic age designs
- Geometric patterns
- Circuit board aesthetics
- Radiation symbols (safe, stylized)
- 1950s sci-fi inspired patterns
- Minimalist abstract designs

### Fallout-Themed Scenes
- Desert wasteland (if applicable)
- Urban decay with retro-futuristic elements
- Vault interiors (abstract)
- Pip-Boy interface mockups
- Terminal/computer aesthetic

## Design Principles

1. **High Contrast**: Text and UI elements must be readable when overlaid
2. **Minimal Distraction**: Not too busy - designed for desktop use
3. **Consistent Style**: All wallpapers should share design language
4. **Scalability**: Look good at all resolutions
5. **Theme Consistency**: Match VaultOS overall aesthetic

## File Naming

Use descriptive names:
- `pipboy-grid-1920x1080.png`
- `vault-door-blue-2560x1440.png`
- `pattern-circuit-3840x2160.png`
- `pipboy-terminal-1920x1080.png`

Format: `{theme}-{description}-{resolution}.{ext}`

## File Formats

- **PNG**: Preferred for graphics with transparency or sharp edges
- **JPG**: Acceptable for photographic-style wallpapers
- **SVG**: Vector wallpapers (less common but scalable)

## Technical Specifications

### Resolution Guidelines
- Aspect ratio: 16:9 (standard) or 21:9 (ultrawide)
- DPI: 72-96 for screen use
- Color space: sRGB

### File Size
- Keep under 5MB per image (optimize)
- Use compression but maintain quality
- Consider progressive JPG for large files

## Required Wallpapers

### Priority 1 (Essential)
1. Pip-Boy interface background (1920x1080 minimum)
2. Vault door (blue/gold variant, 1920x1080 minimum)
3. Simple Pip-Boy green pattern (all resolutions)

### Priority 2 (Recommended)
4. Terminal-style Pip-Boy interface
5. Vault-Tec blue/gold pattern
6. Circuit board pattern
7. Minimalist abstract design

### Priority 3 (Optional)
8. Additional pattern variations
9. Scene-based wallpapers
10. Special variants (dark mode, light accents)

## Creating Wallpapers

### Tools
- **GIMP** - Free raster graphics editor
- **Inkscape** - Vector graphics (for patterns)
- **Blender** - 3D rendering (for complex scenes)
- **ImageMagick** - Batch processing and conversion

### Workflow
1. Create at highest resolution (3840x2160)
2. Design with Pip-Boy/Vault-Tec color palette
3. Export to required resolutions
4. Optimize file sizes
5. Test on different screen sizes
6. Verify readability with desktop icons/text

## Testing

After creating wallpapers:
1. Test at target resolution
2. Verify on different display types (LCD, OLED)
3. Check with desktop icons/text overlaid
4. Ensure colors render correctly
5. Validate file sizes are reasonable

## Contributing

When adding new wallpapers:
1. Follow naming conventions
2. Use correct color palettes
3. Provide multiple resolutions
4. Include source files if possible (SVG, XCF)
5. Document any special considerations
